Differenti modi di uso della libreria

Puoi utilizzare la libreria nei seguenti modi:
  • Utilizzata come Libreria Dinamica (DLL):
    Puoi accedere alla libreria DLL con qualsiasi linguaggio che supporti le DLL create in "C".
    E' questo il caso di compilatori standard C, Microsoft C++, Borland C++, Microsoft C#, Microsoft J#, Microsoft Visual Basic.
    Alcuni esempi sono inclusi con la DLL, in molti casi si tratta di versioni compatibili con l'ultima versione del linguaggio.
    Eccezioni: grazie a collaboratori esterni è possibile utilizzare MediaInfo DLL anche con Microsoft Visual Basic 5 e 6.
  • Utilizzata come ActiveX:
    Grazie a collaboratori esterni esiste un interprete ActiveX.
    Puoi utilizzare MediaInfo in Microsoft Internet Explorer o attraverso Visual Basic Script. Sono disponibili esempi per Internet Explorer e VBS.
  • Utilizzata come libreria statica (LIB):
    Puoi accedere alla libreria con compilatori C/C++ standard.
    librerie per Microsoft C++, Borland C++, Bloodshed DevCpp, MinGW e GCC (Linux) IA32 sono disponibili (in attesa di conferma)
  • Compilando il codice sorgente:
    Puoi compilare il codice sorgente da solo.
    Esistono codici sorgente anche per Microsoft C++, Borland C++, Bloodshed DevCpp, MinGW e GCC (Linux) (in attesa di conferma)

Usa MediaInfo come DLL

Col tuo compilatore puoi accedere alla DLL con alcuni codici.
Ci sono alcuni esempi di inclusi nel pacchetto della DLL, ti consiglio di darci un'occhiata.

Usa MediaInfo come ActiveX

Con un editor HTML o VBS, puoi accedere alla DLL utilizzando alcuni codici.
Ci sono alcuni esempi di inclusi nel pacchetto della DLL (nella cartella "contrib"), ti consiglio di darci un'occhiata.

Compila il codice sorgente

Se hai intenzione di compilare il codice sorgente da solo, hai bisogno di programmi esterni al Progetto MediaInfo, solo con questi potrai compilare il codice di MediaInfo.
Troverai come compilare il codice in questa pagina.
