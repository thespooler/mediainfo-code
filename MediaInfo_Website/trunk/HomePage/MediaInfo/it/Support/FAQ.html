Generali

Qual'è il sito ufficiale?

http://mediainfo.sourceforge.net

OpenCandy stuff is installed!

OpenCandy is used by the Windows GUI installer for providing some Open Source and proprietary software advertisement.
You can look at the OpenCandy provacy policy.
If you don't have OpenCandy screen during the install, or if you don't click on "Yes, Install" (this is NOT the default action) in this screen, nothing is done.
Some files are put in the MediaInfo directory, and some registry keys are in the registry only for uninstallation.
If even with these explanations you don't want OpenCandy stuff, you can:
  • Pay me every month for not having it (I spend a lot of time on MediaInfo, and I try to live from this work)
  • Use the package without installer
  • Use your own installer (this is open source, you can do it)
Keep in mind that this is not an adware installed without your consent.

MediaInfo tries to connect to the Internet!

Yes, it tries (if "Check for newest version" is checked).
When this checkbox is check, MediaInfo connects to mediaarea.eu website too for anonymous statistics. (how often MediaInfo is used, which OS/Language/Country is the most used...)
If you don't want to autorize anonymous statistics, uncheck the "Check for newest version" checkbox.

Su sourceforge ci sono diversi files, quale devo scaricare?

Diepnde da ciò di cui tu hai bisogno:
  • Un'interfaccia grafica : MediaInfo GUI
  • Un programma a riga di comando : MediaInfo Cmd
  • Un DLL necessario ad altri programmi : MediaInfo DLL

Dopo di ciò scegli il sistema operativo:
  • Windows : da Windows 95 a Windows XP
  • Linux : distribuzioni Linux

e la tua piattaforma:
  • 686 : Intel Pentium, AMD Athlon
  • AMD64 : Processori di tipo Intel EMT64, AMD Athlon64

Su sourceforge ci sono diverse versioni, quale devo scaricare?

Ti consiglio sempre di utilizzare l'ultima versione, ma considera che:
  • Le versioni beta potrebbero non essere stabili.
  • Alcune persone potrebbero aver bisogno di versioni specifiche per funzionare con determinati programmi. In questo caso scarica la versione richiesta.

Perché (inserire il nome del formato) non è supportato?

Perché non ho le specifiche tecniche di quel formato.
Se siete in possesso della documentazione relativa a quel formato, vi prego di contattarmi.

Un file non è riconosciuto bene, che faccio?

Sono interessato ad avere il file in questione, vi prego di contattarmi.

Vorrei utilizzare MediaInfo nella mia lingua, perché non lo traducete?

E' ovvio, perché io non parlo la tua lingua! ;-)
Se avete un'ora, potete tradurlo (aprite la fienstra delle opzioni), e quindi inviarmi il file (si trova nella cartella ./plugins/). Non è difficile, Devi solo conoscere un pò di Inglese.

Non trovo la mia domanda tra queste, che faccio?

Puoi avere supporto sul forums.

MediaInfo GUI

Che licenza ha?

MediaInfoGUI utilizza la licenza GPL.

Vorrei compilare il codice sorgente, come faccio?

Puoi scaricare il codice sorgente qui.
Hai bisogno di Borland C++ 6 per compilarlo.
Qui troverai maggiore aiuto nella compilazione.

Perché non esiste una versione GUI di MediaInfo per Linux?

VCL (la libreria grafica della Borland) non è portabile (questa è l'unica), quindi non posso adattare la GUI per Linux.
Se conosci come programmare con wxWidgets (una libreria grafica standard), e sei interessato ad adattare MediaInfo a wxWidgets io sono interessato, ti prego di contattarmi!

MediaInfo DLL

Che licenza ha?

MediaInfoDLL utilizza la licenza LGPL.

Perché la tua DLL non funziona con (inserire il nome del programma)?

Può essere che il programma non supporti l'ultima versione della DLL.
  • Puoi provare a chiedere allo sviluppatore del programma di aggiornarlo
  • Puoi provare ad utilizzare una dele vecchie versioni della DLL

Vorrei utilizzare alcune funzioni della tua DLL, come faccio?

Scarica il pacchetto della DLL, li ci sono diversi esempi in dieversi linguaggi di programmazione (standard C, standard C++, GCC, MinGW, Borland C++ Builder, Microsoft Visual C++, Microsoft Visual Basic, Microsoft C#, Microsoft J#)
Qui trivi l'SDK.

Preferisco le librerie statiche ai DLL, come faccio?

Hai una sola scelta, C++.
Vai nella pagina del download, li troverai un pacchetto con:
  • Librerie compilate con Borland C++ builder 6
  • Librerie compilate con Microsoft Visual C++ 9 (aka 2005). Attenzione: il progetto è collegato alla libreria multithreaded di C++, e le opzioni rispettano lo standard C++ (sta attento, in C++ standard, wchar_t è un tipo, non un short int).
Se hai intenzione di utilizzare un altro compilatore, devi compilare la sorgente.

Vorrei compilare il codice sorgente, come faccio?

Puoi scaricare qui il codice sorgente.
Nota : il codice sorgente ha diverse dipendenzei, potresti quindi incontrare difficoltà nel compilarlo. Qui troverai maggiore aiuto nella compilazione.
