Allgemein

Wie lautet die kanonische Internetadresse der Webseite?

http://mediainfo.sourceforge.net

OpenCandy stuff is installed!

OpenCandy is used by the Windows GUI installer for providing some Open Source and proprietary software advertisement.
You can look at the OpenCandy provacy policy.
If you don't have OpenCandy screen during the install, or if you don't click on "Yes, Install" (this is NOT the default action) in this screen, nothing is done.
Some files are put in the MediaInfo directory, and some registry keys are in the registry only for uninstallation.
If even with these explanations you don't want OpenCandy stuff, you can:
  • Pay me every month for not having it (I spend a lot of time on MediaInfo, and I try to live from this work)
  • Use the package without installer
  • Use your own installer (this is open source, you can do it)
Keep in mind that this is not an adware installed without your consent.

MediaInfo tries to connect to the Internet!

Yes, it tries (if "Check for newest version" is checked).
When this checkbox is check, MediaInfo connects to mediaarea.eu website too for anonymous statistics. (how often MediaInfo is used, which OS/Language/Country is the most used...)
If you don't want to autorize anonymous statistics, uncheck the "Check for newest version" checkbox.

Im SourceForge-Repository gibt es Dateien mit unterschiedlichen Namen, welche soll ich herunterladen?

Das hängt davon ab, was Sie brauchen:
  • Eine grafische Benutzeroberfläche : MediaInfo GUI
  • Eine Kommandozeilenversion : MediaInfo Cmd
  • Eine DLL für ein anderes Programm : MediaInfo DLL

Danach wählen Sie Ihr Betriebssystem:
  • Win : Windows 95 bis Windows XP
  • Lin : Linux-Distributionen

Schließlich wählen Sie Ihre Rechnerarchitektur:
  • 686 : Intel Pentium, AMD Athlon
  • AMD64 : Intel EMT64-fähiger Prozessor, AMD Athlon64

Im SourceForge-Repository gibt es unterschiedliche Versionen, welche soll ich herunterladen?

Immer die neueste Version, ausser:
  • Die Version trägt den Namen "beta" : Sie können die Betaversion herunterladen, aber bedenken Sie dass sie nicht unbedingt stabil ist.
  • Sie werden angewiesen eine spezielle Version herunterzuladen. Nehmen Sie diese Version, vielleicht hat der Entwickler die neue Version nicht validiert.

Warum ist (hier beliebiges Format einsetzen) nicht unterstützt?

Weil mir die Spezifikationen dafür fehlen!
Wenn Sie eine Dokumentation zu diesem Format haben kontaktieren Sie mich bitte.

Eine Datei wird nicht richtig erkannt, was soll ich machen?

Ich habe Interesse an der Datei, bitte kontaktieren Sie mich.

Ich möchte dass MediaInfo in meine Sprache übersetzt wird, warum wird es nicht übersetzt?

Weil ich Ihre Sprache nicht spreche! ;-)
Wenn Sie etwa eine Stunde Zeit erübrigen möchten, können Sie MediaInfo übersetzen (unter Optionen / Voreinstellungen) und mir die Datei (im ./plugins/ Verzeichnis) zusenden. Es ist nicht schwierig, Sie benötigen lediglich ein paar Englischkenntnisse.

Meine Frage ist hier nicht aufgezählt, was soll ich machen?

Sie erhalten im Forum Hilfe.

Die grafische Benutzeroberfläche von MediaInfo (GUI)

Unter welcher Lizenz steht sie zur Verfügung?

MediaInfoGUI ist unter GPL veröffentlicht.

Ich möchte MediaInfo selbst kompilieren, wie geht das?

Sie können den Quellcode auf dieser Seite herunterladen, Sie benötigen Borland C++ 6 um ihn zu kompilieren.
Es gibt eine Hilfe um Abhängigkeiten zu kompilieren.

Warum gibt es keine grafische Benutzeroberfläche von MediaInfo unter Linux?

VCL (die Grafikbibliothek von Borland) ist nicht portabel (dies ist die Einzige), daher kann ich die GUI nicht nach Linux portieren.
Wenn Sie sich in der Programmierung von wxWidgets (eine Standard-Grafikbibliothek) auskennen und interessiert sind MediaInfo an wxWidgets anzupassen schreiben Sie mir bitte!

Kommandozeilenversion von MediaInfo (Cmd)

Unter welcher Lizenz steht sie zur Verfügung?

MediaInfoCmd ist unter GPL veröffentlicht.

Warum gibt es nicht mehr Optionen für die Kommandozeilenversion?

Ich habe mir nicht die Zeit genommen sie zu programmieren, aus Mangel an Interesse von Benutzern.
Sie können einen Vorschlag für neue Funktionen hinzufügen.

Ich möchte MediaInfo selbst kompilieren, wie geht das?

Sie können den Quellcode herunterladen.
Hinweis : Es gibt viele Abhängigkeiten, unter Umständen werden Sie Probleme beim Kompilieren haben. Es gibt eine Hilfe um Abhängigkeiten zu kompilieren.

Die Dynamic Link Library von MediaInfo (DLL)

Unter welcher Lizenz steht sie zur Verfügung?

MediaInfoDLL ist unter LGPL veröffentlicht.

Warum funktioniert die DLL nicht mit (hier beliebiges Programm einsetzen)?

Wahrscheinlich unterstützt das Programm die neueste DLL-Version nicht.
  • Fragen Sie den Entwickler ob er sein Programm anpassen kann
  • Versuchen Sie eine ältere Version der DLL herunterzuladen

Ich möchte die Resultate der DLL in meinem Programm abfragen, wie geht das?

Laden Sie das DLL-Paket herunter, enthalten sind einige Beispiele (C, C++, GCC, MinGW, Borland C++ Builder, Microsoft Visual C++, Microsoft Visual Basic, Microsoft C#, Microsoft J#)
Es gibt auch ein SDK zum herunterladen.

Ich möchte keine DLLs verwenden, ich bevorzuge statische Bibliotheken. Wie mache ich das?

Das ist nur mit C++ möglich.
Auf der Downloadseite gibt es ein Paket mit:
  • der Bibliothek kompiliert in Borland C++ builder 6
  • der Bibliothek kompiliert in Microsoft Visual C++ 9 (aka 2005). Achtung! Das Projekt ist statisch mit der Multithread C++ Bibliothek verlinkt, und die Optionen C++ konform (in Standard-C++ ist wchar_t ein type, keine Integer-Zahl).
Wenn Sie einen anderen Compiler oder andere Optionen verwenden möchten, müssen Sie den Quellcode kompilieren.

Ich möchte MediaInfo selbst kompilieren, wie geht das?

Sie können den Quellcode herunterladen.
Hinweis : Es gibt viele Abhängigkeiten, unter Umständen werden Sie Probleme beim Kompilieren haben. Es gibt eine Hilfe um Abhängigkeiten zu kompilieren.
